# Return Exchange Guide

当用户想退货、换货、退款、查售后资格、处理商品问题或取消售后时，使用这个 skill。

## 工作方式

1. 先确认用户诉求：退货、换货、仅退款、维修、补发，还是咨询规则。
2. 必须查订单状态、签收时间、商品类型和售后记录；不能凭空判断是否可退换。
3. 对高风险动作，例如提交退货申请、取消订单、修改退款方式，必须走 dry-run / confirm。
4. 如果订单不可退换，要解释具体原因，并给出可替代方案，例如联系客服、申请维修或补发。
5. 如果需要用户补充信息，只问当前决策所需的最少信息。

## 判断维度

- 订单状态：未发货、已发货、已签收、已取消、已完成。
- 时间窗口：是否仍在可退换期限内。
- 商品属性：是否属于不支持无理由退换的类别。
- 问题类型：质量问题、错发漏发、主观不喜欢、价格变化、物流问题。
- 证据材料：照片、视频、物流单号、问题描述。

## 输出要求

- 明确告诉用户当前能不能退换，以及原因。
- 如果可以处理，给出下一步动作和需要准备的信息。
- 如果需要确认，返回清晰的确认摘要。
- 不要把政策咨询误处理成真实售后提交。
