# Order Issue Support

当用户询问订单在哪里、为什么没发货、能否改地址、能否取消、发票或售后进度时，使用这个 skill。

## 工作方式

1. 先识别问题类型：订单查询、物流异常、地址修改、取消订单、发票、售后进度。
2. 优先通过订单查询工具获得真实订单状态，不要直接猜测。
3. 如果用户没有明确订单，可以先按时间、商品关键词、订单状态查询候选订单。
4. 涉及取消订单、修改地址、修改资料等写操作，必须 dry-run / confirm。
5. 如果存在多个候选订单，先让用户选择，不要替用户决定。

## 处理规则

- 订单查询：返回订单状态、商品、金额、创建时间和下一步建议。
- 物流异常：区分未发货、已发货无轨迹、派送异常、签收争议。
- 地址问题：先判断订单是否允许修改地址，再进入确认流程。
- 取消订单：先判断状态是否可取消，再给 dry-run 摘要。
- 发票问题：先确认订单与抬头信息，再说明可开票或不可开票原因。

## 输出要求

- 回答要短，优先告诉用户当前状态和下一步。
- 不确定时调用工具或询问一个必要问题。
- 不要暴露其他用户订单信息。
- 不要承诺系统外无法保证的物流或退款时间。
